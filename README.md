# Spring PetClinic Test Metrics Fork 

Этот репозиторий — форк spring-petclinic, в который добавлено расширение для JUnit, собирающее метрики выполнения тестов и экспортирующее их в формате, пригодном для Prometheus. Необходимоо для мониторинга качества тестов, времени выполнения и трендов в CI/CD.

!(Grafana example)[grafana-example.png]

Описание
--------
Расширение JUnit собирает данные о:
- кол-ве запущенных тестов,
- успехах/провалах/пропущенных,
- длительности тестов,
- помеченных тестах (по тегам/именам).

Данные экспортируются в формат, который может считывать Prometheus (экспозиция через HTTP-эндоинт /metrics). После этого можно строить дашборды в Grafana.

Возможности
----------
- Инструментирование тестов без изменения существующей логики (через JUnit Extension).
- Экспорт метрик в Prometheus (локальный HTTP-экспортер).
- Поддержка агрегатов по suite/class/tag.
- Простая конфигурация через переменные окружения или system properties.

Как это работает
----------------
Расширение регистрирует слушателей на старте каждого теста и после его завершения. По завершении теста обновляются соответствующие счётчики и гистограммы/summary. В проекте запускается лёгкий HTTP-сервер (или используется встроенный экспортер), который отдаёт метрики на /metrics для Prometheus.
